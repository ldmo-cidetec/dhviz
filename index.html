DHViz: A Denavit-Hartenberg Visualizer

Help

DHViz is an interactive web-based tool for configuring and visualizing the kinematics of open-chain robotic manipulators using the Denavit-Hartenberg convention (Craig formulation). It computes transformation matrices, the geometric Jacobian, and exports ready-to-use code in Python (NumPy/SymPy) and MATLAB.
  1. Open the DH tab.
  2. Press Add link for each link of your robot.
  3. Select the joint type: R (revolute), P (prismatic), or F (fixed). The last link is always fixed.
  4. Enter the four geometric parameters for each link: \alpha_{i-1},\, a_{i-1},\, d_i,\, \theta_i.
  5. The 3D visualization updates automatically.
Each row i defines the transformation {}^{i-1}T_i with four parameters:

  • \alpha_{i-1} - angle from Z_{i-1} to Z_i about X_{i-1}
  • a_{i-1} - distance from Z_{i-1} to Z_i along X_{i-1}
  • d_i - distance from X_{i-1} to X_i along Z_i
  • \theta_i - angle from X_{i-1} to X_i about Z_i

The variable parameter is \theta_i for revolute joints and d_i for prismatic joints.
Revolute (R) and prismatic (P) joints have a slider labeled q_i in the Joint variables section below the table. Drag it to move the joint and see the robot update in real time.

The min and max inputs at the ends of each slider let you set the joint limits. To set a precise value, type it directly in the input field on the right of the slider.
The Matrices tab shows two columns side by side for each link:

  • Symbolic (blue) - the general form with q_i as symbols.
  • Numeric (gray) - evaluated with the current joint values.

At the bottom, the accumulated transformation {}^0T_n = {}^0T_1 \cdot \ldots \cdot {}^{n-1}T_n is shown numerically.
The Jacobian tab shows the geometric Jacobian J(q) \in \mathbb{R}^{6 \times n}, which relates joint velocities to end-effector velocities:

\begin{bmatrix}v\\\omega\end{bmatrix} = J(q)\,\dot{q}
The manipulability measure w = \sqrt{\det(J_v J_v^T)} indicates how far the robot is from a singular configuration. A value near zero means the robot is close to losing a degree of freedom.
Reference points (e.g. center of mass positions) can be added in the DH tab under Reference points.

Each point has a label, a frame \{i\} it belongs to, and local coordinates x, y, z. It is displayed as a red sphere in the 3D viewer and its position in the base frame \{0\} is computed as {}^0p = {}^0T_i \cdot {}^ip_{\text{local}} in the Matrices tab.
The Export tab generates ready-to-run code for your robot.

Use the Numeric / Symbolic toggle per language:

  • Numeric - uses NumPy (Python) or native MATLAB arrays.
  • Symbolic - uses SymPy (Python) or the Symbolic Math Toolbox (MATLAB), with q_i as symbolic variables.

The generated code includes forward kinematics, frame transforms, the geometric Jacobian, and the positions of all reference points. Press Copy to copy to clipboard.
The URDF tab generates a Unified Robot Description Format file for use in ROS, RViz, MoveIt, and other robotics frameworks.

The file includes:

  • A base_link at the world origin \{0\}.
  • One <joint> and <link> per row, with origin computed from \alpha_{i-1}, a_{i-1}, d_i, \theta_i.
  • Type revolute or prismatic for actuated joints, fixed for passive.
  • Joint limits from the slider min/max values.
  • Cylinder visuals connecting consecutive frames with spheres at each joint origin.
  • A red sphere at the end-effector.

Use Download .urdf to save, or Copy to copy to clipboard.
The 3D viewer uses OrbitControls:

  • Rotate: left click and drag.
  • Zoom: scroll wheel or pinch on touch.
  • Pan: right click and drag.

Use the View panel (top-left of the viewer) to switch to preset orientations: ISO (isometric), TOP (from +Z), FRONT (from +Y), and RIGHT (from +X).

Use the Scale panel (bottom-right of the viewer) to adjust the size of links, joints, axes, and labels independently.
